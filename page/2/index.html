css水平与垂直居中和多列布局

css布局之水平垂直居中和多列布局

  • 水平垂直居中

    子元素于父元素垂直及水平居中且其（子元素与父元素）高度宽度均可变。

  • inline-block + text-align + table-cell + vertical-align

此方法的好处在于兼容性好。

  • absolute + transform

此方法的好处在于绝对定位脱离文档流，不会对后续元素的布局造成影响，但transform 为 CSS3 属性，有兼容性问题。

  • flex + justify-content + align-items

此方法的好处在于只需设置父节点属性，无需设置子元素,但存在兼容性问题。

多列布局

多列布局在网页中非常常见（例如两列布局），多列布局可以是两列定宽，一列自适应， 或者多列不定宽一列自适应还有等分布局等。

  • 一列定宽，一列自适应

  • float + margin

此方法IE 6 中会有3像素的 BUG，解决方法可以在 .left 加入 margin-left:-3px。

  • float + margin + (fix) 改造版

此方法不会存在 IE 6 中3像素的 BUG，但 .left 不可选择， 需要设置 .left {position: relative} 来提高层级。 此方法可以适用于多版本浏览器（包括 IE6）。缺点是多余的 HTML 文本结构。

  • float + overflow

此方法在设置 overflow: hidden 会触发 BFC 模式（Block Formatting Context）块级格式化文本。BFC 中的内容与外界的元素是隔离的。优点在于样式简单，但不支持IE6。

  • table

table 的显示特性为每列的单元格宽度合一定等与表格宽度。 table-layout: fixed; 可加速渲染，也是设定布局优先。注意：table-cell 中不可以设置 margin 但是可以通过 padding 来设置间距。

  • flex

flex-item 默认为内容宽度。在低版本浏览器兼容问题，存在性能问题，只适合小范围布局。

  • 两列定宽，一列自适应

多列定宽的实现可以更具单列定宽的例子进行修改与实现。

  • flex + justify-content

这种方法的好处在于只需设置父节点属性，无需设置子元素，但此种方法存在兼容性问题。

3、如果你是想修改别人的代码，只需在对应的项目中点击“Fork”按钮，然后就可以看到该项目在自己的账号中了。

一般情况我们还需要将代码下载到本地，用编译工具去修改，这就有同步代码到网上仓库的问题，很多人介绍使用msysgit和它的客户端，但是我弄了好久都没有弄好(配置啊，SSH Key啊，翻墙下载啊，奔溃了)。后来用desktop才弄好。

四、Desktop 的使用

1、点开左上角的加号，创建一个仓库

注意：①这里需要注意的下载网上的代码，点看Clone，选着你的账户，会出现你网上仓库中的项目，选择一个，点击Clone repository按钮即可。

2、创建仓库完成

创建仓库完成后在本地 文件夹下会生成一些系统文件。现在你在仓库下添加文件，修改文件等操作都会被本地github客户端监测，在客户端右上角的changes中可以看到提示。点击这个提示，弹出提交框。

PHP学习

一、什么是PHP？

PHP是一门后台编程语言，是目前比较主流的后台编程语言。

PHP优点：
1、开源
2、能运行于各种平台
3、兼容几乎所有的服务器
4、语法简单，极易学习
5、功能强大，能做大型开发
6、能够动态生成web页面内容（php代码可以嵌入到html代码中）

二、PHP环境搭建

Apache简介

  • Apache是Web服务器的软件
  • 它可以运行在几乎所有的计算机平台上面
  • 这是最流行的Web服务器软件之一
  • Apache的官网地址：http://www.apache.org

MySQL简介

  • MySQL是关系型数据库管理系统
  • 拥有体积小，速度快，开放源码等有优点
  • Apache的官网地址：http://www.mysql.com

XAMPP简介

  • XAMPP是一个功能强大的建（Apache+MySQL+PHP+Perl）软件站集成软件包
  • 引入XAMPP的目的：手动安装Apache+MySQL+PHP环境过于复杂，而XAMPP帮我们集成了这个环境，只需要点击安装即可
  • Apache的官网地址：http://www.mysql.com

XAMPP常见问题

  • apache启动失败，运行sudo apachectl stop
  • MYSQL启动失败：
  • 第一步：sudo/Library/Startupltems/MySQLCOM/MySQLCOM stop
  • 第二步：使用ps-eflgrep mysql查找进程号，通过kill杀掉进程

3.php注释：

  • 变量以$开头，后面跟变量名
  • 变量名只能由字母，数字，下划线组成，不能以数字开头
  • 变量名区分大小写

  • 引用赋值：创建的一个变量与另一个变量引用的内容相同

5.超全局变量：

  • php提供了很多预定义的变量，用于提供大量与环境有关的信息

  • 打印/输出超全局变量：print_r($_SERVER)
